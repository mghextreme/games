# Game Rooms

A multiplayer game room platform where users can create, search, and join rooms to play various games with friends in real-time.

## Features

- **Room Management**: Create, search, and join game rooms
- **Password Protection**: Optional passwords for private rooms
- **Real-time Updates**: Live player lists and game state synchronization via Convex
- **Multi-Game Support**: Extensible architecture for adding new games
- **Guest-only Play**: No registration required - everyone plays as a guest
- **Reconnection**: Players can reconnect within 5 minutes if disconnected
- **Auto-Cleanup**: Inactive rooms are automatically deleted after 12 hours

## Tech Stack

- **Frontend**: Nuxt 3 (Vue 3, TypeScript)
- **UI**: shadcn-vue + Tailwind CSS
- **Backend**: Convex (self-hosted, real-time database)
- **Deployment**: Docker Compose (all services run locally)

## Quick Start

### Prerequisites

- Docker and Docker Compose installed
- Node.js 18+ (for development)

### Running with Docker (Recommended)

1. Start all services:
   ```bash
   docker compose up
   ```

2. Generate an admin key for Convex (first time only):
   ```bash
   docker compose exec convex-backend ./generate_admin_key.sh
   ```

3. Create a `.env` file in the project root:
   ```bash
   CONVEX_SELF_HOSTED_URL='http://127.0.0.1:3210'
   CONVEX_SELF_HOSTED_ADMIN_KEY='<your admin key from step 2>'
   ```

4. Deploy the Convex functions:
   ```bash
   pnpm convex dev
   ```

5. Access the application:
   - **Frontend**: http://localhost:3000
   - **Convex Dashboard**: http://localhost:6791
   - **Convex Backend**: http://localhost:3210

### Development Mode

For local development with hot reload:

1. Start the Convex backend:
   ```bash
   docker compose up convex-backend convex-dashboard
   ```

2. Generate admin key and create `.env` (see steps 2-3 above)

3. Run Convex dev (watches for changes):
   ```bash
   pnpm convex dev
   ```

4. Start the frontend (in a separate terminal):
   ```bash
   cd frontend
   pnpm install
   pnpm run dev
   ```

5. Visit http://localhost:3000

### Running Tests

```bash
cd frontend
pnpm test
```

For interactive test UI:
```bash
pnpm run test:ui
```

## Adding New Games

See [AGENTS.md](./AGENTS.md) for the game development guide.

1. Create a new directory: `frontend/app/lib/games/{game-id}/`
2. Implement `types.ts` with state and move types
3. Implement `engine.ts` with game logic functions
4. Create the Vue component in `frontend/app/components/games/{game-id}/`
5. Register the game in `frontend/app/lib/games/registry.ts`

## Configuration

### Environment Variables

**For Convex CLI (in `.env`):**
- `CONVEX_SELF_HOSTED_URL`: Convex backend URL (default: `http://127.0.0.1:3210`)
- `CONVEX_SELF_HOSTED_ADMIN_KEY`: Admin key generated by the backend

**Frontend (in docker-compose.yml):**
- `NUXT_PUBLIC_CONVEX_URL`: Convex backend URL

### Services

| Service | Port | Description |
|---------|------|-------------|
| Frontend | 3000 | Nuxt application |
| Convex Backend | 3210 | Database and functions |
| Convex HTTP Actions | 3211 | HTTP action endpoints |
| Convex Dashboard | 6791 | Admin UI for debugging |

### Data Persistence

Convex data is stored in a Docker volume (`convex_data`). To reset:
```bash
docker compose down -v
```

## Guest Identity

- Every user is a guest identified by a UUID stored in localStorage
- Guest ID persists across browser sessions (same browser = same identity)
- Display name can be changed anytime from the header
- If localStorage is cleared, the user gets a new identity
